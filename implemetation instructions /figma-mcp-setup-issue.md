# figma-mcp: Binary Not Executable After `npm link`

## Problem

After running `npm link` in the `figma-mcp` project directory, the `figma-mcp` command is not found when trying to use it (e.g., from Claude Code's MCP server config).

```
Failed to reconnect to figma-bridge.
```

## Root Cause

The `package.json` `bin` field points to `./dist/server/index.js`:

```json
{
  "bin": {
    "figma-mcp": "./dist/server/index.js"
  }
}
```

After `npm link`, the symlink is correctly created in the global bin directory:

```
figma-mcp -> ../lib/node_modules/figma-mcp/dist/server/index.js
```

However, `dist/server/index.js` is generated by `tsc` without the execute permission:

```
-rw-r--r--  dist/server/index.js   # missing +x
```

Even though the file has the correct shebang (`#!/usr/bin/env node`), the OS cannot execute it without the `x` permission bit.

## Fix

Run `chmod +x` on the built output:

```bash
chmod +x dist/server/index.js
```

## Suggested Permanent Fix

Add a `postbuild` script in `package.json` to automatically set the permission after every build:

```json
{
  "scripts": {
    "build": "tsc",
    "postbuild": "chmod +x dist/server/index.js"
  }
}
```

This ensures the binary is always executable after `npm run build`, so `npm link` will work correctly.
